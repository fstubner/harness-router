Local MCP router for coding agents

Stop manually switching coding agents.

Harness Router is a local MCP server that routes each coding task through Claude Code, Codex, Cursor, Gemini CLI, opencode, Copilot CLI, or an API fallback based on your priority list and live route health.

What it does

One MCP entry point for every coding route.

Your host calls one tool. Harness Router walks your model priority list, tries subscription-backed CLIs first, skips broken routes, and falls back to API only when the local options are dry.

01

Write the policy once

List the models you prefer and the routes that can serve them. The host does not need to know which vendor wins a given request.

02

Use subscription quota first

Claude Code, Codex, Cursor, opencode, Copilot CLI, and Gemini CLI can be tried before metered OpenAI-compatible endpoints.

03

See the route decision

Status resources and the dashboard show route reachability, quota, circuit breakers, and the route chosen for a task.

Model-first configuration

Configure the outcome you want, then attach the routes that can deliver it. This keeps the policy readable when you add another CLI, account, local model, or API fallback. 

# ~/.harness-router/config.yaml
priority:
  - claude-opus-4-7
  - gpt-5.4

models:
  claude-opus-4-7:
    subscription:
      - harness: claude_code
      - harness: opencode
    metered:
      base_url: https://api.anthropic.com/v1
      api_key: "${ANTHROPIC_API_KEY}"
  gpt-5.4:
    subscription:
      - harness: codex
      - harness: copilot
// MCP tool surface
code({
  prompt: "Refactor the auth callback",
  mode: "single",
  hints: { model: "claude-opus-4-7" }
})

code({
  prompt: "Compare two migration plans",
  mode: "fanout",
  models: ["gpt-5.4", "claude-opus-4-7"]
})

Small MCP surface

Hosts see one coding tool: code. Use single mode for normal work and fanout mode when you want multiple model routes to answer the same prompt. Status lives in resources, not extra tools.

Compared to opencode

opencode is an agent. Harness Router is the routing layer.

They can work together. opencode remains one of the execution routes. Harness Router decides when opencode, Codex, Claude Code, Cursor, a generic CLI, or an API route should handle the next request.

Runtime surface

Routing state stays inspectable.

The server exposes one tool for work and two resources for state. This keeps MCP clients clean while still giving operators a way to see what is reachable.

code
Dispatch a coding task in single-route mode or fanout mode.
harness-router://status
Text status for configured models, route reachability, quota, and breakers.
harness-router://status.json
Machine-readable status for scripts, hosts, and diagnostics.
Get started

Install once, then wire it into your MCP host.

Requires Node 20 or newer and at least one configured route: Claude Code, Codex, Cursor agent, Gemini CLI, opencode, Copilot CLI, a generic CLI, or an OpenAI-compatible API endpoint.

FAQ

Questions before you put a router in front of your agents.

Is Harness Router an AI coding agent?

No. Harness Router is a local MCP router. It delegates coding work to tools such as Claude Code, Codex, Cursor, Gemini CLI, opencode, Copilot CLI, generic CLI routes, and OpenAI-compatible APIs.

How is Harness Router different from opencode?

opencode is a coding agent you use directly. Harness Router sits above coding agents and decides which route should handle a task based on model priority, subscription availability, quota, circuit breaker state, and metered fallback.

Does Harness Router send code to a hosted service?

No hosted Harness Router service is required. It runs locally as an MCP server and dispatches only to the CLIs or APIs you configure.

Which tools can Harness Router route to?

Claude Code, Codex, Cursor agent, Gemini CLI, opencode, GitHub Copilot CLI, generic CLI routes, and OpenAI-compatible API endpoints.

What happens when a route hits a limit?

Harness Router marks that route with a circuit breaker and moves to the next eligible route. Status resources and the dashboard show what happened.
