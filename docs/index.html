The framing

The harness is half the answer.

A model alone doesn’t edit your codebase. The harness does — Cursor’s workspace indexer, Claude Code’s Bash and Read tools, Codex’s --full-auto loop, Gemini’s 2M-token context. The same weights behave differently behind different scaffolds.

Harness Router scores every (model × harness) pair as a first-class service. Your task lands in the harness whose tooling actually fits the work, not just the harness whose underlying model tops a leaderboard.

How it picks

Five signals, one decision.

For every dispatch, the router scores each enabled service and routes to the highest live score. No coin-flips, no round-robin.

  1. quality_score Normalised ELO × thinking multiplier. Source priority: bundled benchmark blend (Arena + Aider + SWE-bench) → live Arena code leaderboard → conservative default.
  2. cli_capability How much the harness adds beyond raw model. Reference points: claude_code 1.10, codex 1.08, cursor 1.05, gemini_cli 1.00.
  3. capabilities[task_type] Per-service relative strength on execute / plan / review. Cursor crushes execute; Opus wins plan; Gemini holds for long-context review.
  4. quota_score Live availability ∈ [0, 1], updated from each dispatch response. When a free tier exhausts, traffic flows away from it.
  5. circuit_breaker Failing services trip out of rotation automatically. Recovery is half-open + probed — no more cascading retry storms.
MCP surface

Twelve tools your client gets for free.

Drop the server into Claude Desktop, Cursor, or any MCP client. The routing logic, leaderboard cache, quota tracking, and circuit breakers are wrapped behind twelve plain tools — and any third-party CLI you register via YAML routes through code_auto with the harness hint.

code_auto
Pick the best available service for the prompt. Hints: task_type, harness, prefer_large_context, service.
code_mixture
Fan out to multiple services in parallel. Returns every output for synthesis — use it on architecture decisions.
code_with_claude
Force any enabled claude_code harness service.
code_with_cursor
Force any enabled cursor harness service.
code_with_codex
Force any enabled codex harness service.
code_with_gemini
Force any enabled gemini_cli harness service.
code_with_opencode
Force any enabled opencode harness service. Provider-agnostic OSS agent with multi-subscription support.
code_with_copilot
Force any enabled copilot harness service. The standalone @github/copilot agentic CLI — GitHub-tooling-aware (gh, repo, PR context).
get_quota_status
JSON summary of quota usage and circuit-breaker state per service.
list_available_services
Which services are enabled and CLI-reachable on this host.
dashboard
Formatted overview of services, tiers, ELO, quota, and breaker state.
setup
Guided check of CLI auth, quota access, and config validity.
Install

Two commands and a config block.

Requires Node 20+ and at least one installed CLI: claude, codex, gemini, Cursor’s agent, opencode, copilot (GitHub Copilot CLI), or any third-party CLI you register via YAML. Authenticate the ones you want to use, then drop the server into your MCP client.

1. Install

# One-shot — no global install
npx -y harness-router-mcp mcp

# Or install globally
npm install -g harness-router-mcp
harness-router-mcp --help

2. Wire into Claude Desktop / Cursor

{
  "mcpServers": {
    "harness-router": {
      "command": "npx",
      "args": ["-y", "harness-router-mcp", "mcp"],
      "env": {
        "HARNESS_ROUTER_CONFIG": "/path/to/config.yaml",
        "GEMINI_API_KEY": "your-key-here"
      }
    }
  }
}

3. Optional — tune defaults

# config.yaml — zero config also works; this just tunes auto-detect.
overrides:
  claude_code: { weight: 1.2 }
  gemini_cli:  { thinking_level: medium }

disabled: [cursor]    # skip an installed CLI

Restart your MCP client. The twelve tools above are live in chat — and on the CLI as harness-router-mcp dashboard --watch.
