The point

Subscriptions you've paid for, used to the limit.

You're already paying $20/month for Claude Pro. Maybe another $20 for ChatGPT Plus. Maybe Cursor on top. Each one comes with a flat-rate quota that resets every few hours. When you hit a limit on one of them, the natural next move is to switch to another tool you're paying for — not to swipe a credit card for metered API access at $5–15 per million tokens.

But you don't actually do that. You hit the limit, wait it out, or silently start paying API rates because you forgot to switch. Harness Router automates the switch. You declare a model preference list. It walks down it, picking the highest-priority model that has a working route — preferring subscription-backed CLIs and only touching metered API as a last resort.

How it works

One algorithm, four lines.

Walk the model priority list. For each model, try every subscription route first. Then try metered. Drop to the next model only when both tiers are exhausted. 

# config.yaml
model_priority:
  - claude-opus-4.7
  - gpt-5.4
  - claude-sonnet-4.6
  - gemini-3.1-pro

services:
  claude_code:    { harness: claude_code,    model: claude-opus-4.7,    tier: subscription }
  cursor:         { harness: cursor,         model: claude-sonnet-4.6,  tier: subscription }
  codex:          { harness: codex,          model: gpt-5.4,            tier: subscription }
  copilot:        { harness: copilot,        model: gpt-5.4,            tier: subscription }
  gemini_cli:     { harness: gemini_cli,     model: gemini-3.1-pro,     tier: subscription }
  anthropic_api:  { type: openai_compatible, model: claude-opus-4.7,    tier: metered,
                    base_url: https://api.anthropic.com/v1, api_key: "${ANTHROPIC_API_KEY}" }

With this config, an Opus task tries Claude Code first. If your Claude Pro quota is dry, it falls through to the Anthropic API. Only when Anthropic is also dry does the router drop to GPT-5 and try Codex / Copilot. The model is what you care about; the route is plumbing.

MCP tools

Four tools your MCP client picks up automatically.

Add the server to Claude Desktop, Cursor, or any MCP client. You get four tools that handle routing, quota tracking, and circuit breaking for you. Most coding tasks just call code.

code
Walk your model priority list and dispatch to the best available route. Hints: model (bump a model to the front) or service (force a specific dispatcher).
code_mixture
Fan out to every available service in parallel. One result per service; you synthesise. Useful for architecture decisions where multiple perspectives help.
dashboard
Multi-line text status — model priority, per-service reachability, quota, breaker state, the router's current pick.
get_quota_status
JSON version of dashboard, for tooling.
Install

Two commands and a config block.

You need Node 20 or newer and at least one CLI installed: claude, codex, gemini, Cursor’s agent, opencode, copilot (the GitHub Copilot CLI), or anything you register yourself in YAML. Log in to the ones you want to use, then add the server to your MCP client.

1. Install

# One-shot — no global install
npx -y harness-router-mcp mcp

# Or install globally
npm install -g harness-router-mcp
harness-router-mcp --help

2. Wire into Claude Desktop / Cursor

{
  "mcpServers": {
    "harness-router": {
      "command": "npx",
      "args": ["-y", "harness-router-mcp", "mcp"],
      "env": {
        "HARNESS_ROUTER_CONFIG": "/path/to/config.yaml",
        "GEMINI_API_KEY": "your-key-here"
      }
    }
  }
}

3. Optional — tune defaults

# config.yaml — zero config also works; this just tunes auto-detect.
overrides:
  claude_code: { weight: 1.2 }
  gemini_cli:  { thinking_level: medium }

disabled: [cursor]    # skip an installed CLI

Restart your MCP client. The four tools above are live in chat. You can also watch service status from the terminal with harness-router-mcp dashboard --watch.
